1. Встановити та налаштувати Apache Tomcat.

2. Створити сервлет, який буде виконувати CRUD операції над наступними сутностями:
    - Реалізувати клас Order. Цей клас зберігатиме значення: id, date, cost, products;
    - Реалізувати клас Product. Цей клас зберігатиме значення: id, name, **cost**;

3. Що має вміти сервлет:
    - Створення замовлення із продуктами;
    - Отримання замовлення по id;
    - Оновлювати замовлення;
    - Видаляти замовлення по id;

CRUD - create, read, update, delete. Кожен метод сервлету має
відповідати одній із CRUD-операцій (

    create - POST,
    read - GET,
    update - PUT,
    delete - DELETE).

Примітка, для того, щоб парсити вхідні та вихідні запити з/у json формат,
можна використовувати бібліотеку jackson

4. Написати юніт-тести на сервлет із використанням Mockito + **JUnit 5**

Домашку здаємо через git.

Спосіб 1: створюємо новий репозиторій. В цьому репозиторії від гілки main ствоюємо нову
гілку і переключаємось на неї. Зробили домашку і робимо пуш в новостворену гілку.
Відкриваємо GitHub в браузері і робимо пулл реквест. Посилання на створений пулл
реквест скидаємо на перевірку.

Спосіб 2: у вас уже є репозиторій для домашки і ви не плануєте створювати ще один.
В цьому репозиторії від гілки main створюємо нову гілку, переключаємося на неї,
робимо домашку, зробили - робимо пуш в новостворену гілку. Йдемо на GitHub і створюємо
пулл реквест. Посилання на створений пулл реквест свидаємо на перевірку.
